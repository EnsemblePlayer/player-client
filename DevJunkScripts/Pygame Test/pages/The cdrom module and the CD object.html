The cdrom module and the CD object

First, we import the module - we have to do that.
>>> import pygame.cdrom

Second, we initialize the module - we have to do that also.
>>> pygame.cdrom.init()

Now we can find out how many cd-rom drives our computer has:
>>> pygame.cdrom.get_count()
2
Wait a minute. My computer has a dvd-rom drive and a cdrw-drive. Oh I see. For pygame all the drives our computer has are cd-rom drives.

To find out if the cdrom module is initialized we write:
>>> pygame.cdrom.get_init()
1
The above function will return 1 if true and 0 if false.

And to uninitialize the cdrom module we write:
>>> pygame.cdrom.quit()
-------
Q: So what is the cdrom module good for?
An: Answer is that it is needed so to create a CD object.

Q: And what is this CD object?
An: A CD object is an object (like in object oriented programming) that represents a cd-rom drive.

Q: And why do you want to create a CD object?
An: Because you want to actually access the cd-rom drive in order to hear some music.

Thus we write something like:
>>> pygame.cdrom.init()
>>> a = pygame.cdrom.CD(1)
a represents now the second cd-rom drive in my computer. If I wanted to access the first cd-rom drive in my computer or more likely if my computer had only one cd-rom drive, I would have to write:
>>> a = pygame.cdrom.CD(0)
I think you get the picture.

Q: "OK you smart ass" (you'd ask) "can we tell which cd-rom drive is 0 and which cd-rom drive is 1?"
An: Answer is simple. We can't. We have to ask the computer which one's which. But we can also ask python:
>>> a.get_name()
'D:\\'
And we can always ask back which cd-rom drive is represented by a:
>>> a.get_id()
0

To access the rest of the functions the CD object provides, we have to initialize it. In our case we write:
>>> a.init()
Now we can listen to some music. Put an audio CD in the cd-rom drive you just initialized and type something like:
>>> a.play(0)
And it actually plays it! It's not a miracle but it is python and pygame playing a CD. I mean, think about it!
The 0 inside the parentheses stands for the first track of our audio CD. Remember: only audio tracks. That means no mp3 :-(
Well, at least for the moment.

import pygame.cdrom
pygame.cdrom.init()

a = pygame.cdrom.CD(0)
a.init()

n = a.get_numtracks()

for c in range(n):
    a.play(c)
    while a.get_busy():
        pass              

a.quit()
a = None
pygame.cdrom.quit()

Explanation
I really don't think there is any need for explanation here except that if we omit lines 13 and 14, python will only play the last track of the CD.
If you want just to check out this program or check out a CD, be sure that line 12 becomes: a.play(c,0,5). That way you will only hear the first five seconds of every track and not the whole CD.
Also be sure to run this program from the IDLE. If you decide to quit before the CD finishes, it's best to manually uninitialize (lines 16-18) after the Ctrl-C.
